## Artifact

<Callout type="info" title="Stable object, changing versions">
  An artifact is the permanent object people share. Versions are the immutable
  content snapshots attached to it over time.
</Callout>

An artifact is a stable logical object with one owner, one project, one slug, and one render type. It is the thing people share: a Markdown report, HTML page, JSX prototype, review surface, explainer, or one-off editor.

The public path is:

```txt
/{ownerUsername}/{projectSlug}/{artifactSlug}
```

## Version

Every create or update writes a new immutable version. Versions are content-hashed, monotonically numbered, and parent-linked so humans and agents can inspect changes over time.

Version history powers:

<Cards>
  <Card title="Diff" description="Compare two snapshots before sharing or accepting an agent edit." />
  <Card title="Restore" description="Create a new head version from an older immutable snapshot." />
  <Card title="Audit" description="Review who created, restored, or changed an artifact over time." />
</Cards>

## Render Type

Artifacts currently support:

<Tabs items={["Markdown", "HTML", "JSX/TSX"]}>
  <Tab>
    Specs, reports, checklists, research notes, implementation plans, and PR
    writeups.
  </Tab>
  <Tab>
    Rich standalone reports, visual pages, dashboards, and explainers that need
    safe previewing.
  </Tab>
  <Tab>
    Interactive Preact-backed prototypes, throwaway editors, and one-use tools.
  </Tab>
</Tabs>

Each render path is treated as untrusted generated content and is displayed through controlled preview surfaces.
