## Publishing flow

<Steps>
  <Step>
    ### Pick a project

    Publish into a stable project slug so related outputs are easy to find:

    ```bash
    artifacts project create --json '{"ownerUsername":"alice","slug":"research","title":"Research"}'
    ```
  </Step>

  <Step>
    ### Push a file

    <Tabs items={["Markdown report", "Interactive prototype"]}>
      <Tab>
        ```bash
        artifacts push --owner alice --project-slug research --file ./summary.md
        ```
      </Tab>
      <Tab>
        ```bash
        artifacts push --owner alice --project-slug research --file ./prototype.tsx --title "Filter prototype"
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step>
    ### Use explicit metadata when filenames are not enough

    ```bash
    artifacts push \
      --owner alice \
      --project-slug research \
      --file ./final.md \
      --slug q2-market-map \
      --title "Q2 Market Map"
    ```
  </Step>

  <Step>
    ### Avoid accidental prompts in automation

    ```bash
    export AGENT_ARTIFACTS_NO_INPUT=1
    export AGENT_ARTIFACTS_TOKEN="..."
    ```

    Missing required inputs then fail fast instead of opening a browser login
    flow.
  </Step>
</Steps>

<Callout type="success" title="Result">
  The artifact URL stays stable. Re-publishing with the same slug appends a new
  immutable version instead of replacing history.
</Callout>
