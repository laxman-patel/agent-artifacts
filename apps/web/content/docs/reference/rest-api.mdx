## Route Groups

<Tabs items={["Auth", "Profile", "Projects", "Artifacts", "Sharing"]}>
  <Tab>
    ```txt
    GET|POST /api/auth/*
    GET /.well-known/oauth-protected-resource
    GET /.well-known/oauth-authorization-server
    ```

    OAuth metadata supports MCP clients and other OAuth-aware integrations.
  </Tab>
  <Tab>
    ```txt
    GET /api/profile/me
    GET /api/profile/projects
    GET /api/profile/artifacts
    ```

    Discover the current principal and visible resources.
  </Tab>
  <Tab>
    ```txt
    POST /api/projects
    GET /api/by-path/:ownerUsername/:projectSlug
    GET /api/slug-preview/:username/:projectSlug/:slug
    ```

    Projects group artifacts and provide the middle segment of artifact URLs.
  </Tab>
  <Tab>
    ```txt
    POST /api/artifacts
    GET /api/artifacts/:id
    GET /api/artifacts/:id/content
    POST /api/artifacts/:id/versions
    POST /api/artifacts/:id/restore
    GET /api/by-path/:ownerUsername/:projectSlug/:slug
    ```

    Create the logical artifact, append immutable versions, read content, and
    resolve human URLs.
  </Tab>
  <Tab>
    ```txt
    POST /api/artifacts/:id/share-links
    GET /api/artifacts/:id/share-links
    DELETE /api/share-links/:id
    GET /api/audit-events
    ```

    Use these routes for collaboration controls and review trails.
  </Tab>
</Tabs>
