## Design Goals

<Callout type="info" title="Agent-first command design">
  The `artifacts` CLI is a deterministic wrapper over the REST API. It is built
  for agents first: named flags, schema discovery, stable JSON envelopes, clear
  exit codes, and no hidden prompts when `--no-input` is enabled.
</Callout>

## Discover Commands

Agents should use schema discovery instead of parsing help output:

```bash
artifacts schema
```

The schema includes command names, HTTP method/path mappings, request body JSON Schemas, examples, global flags, and output contracts.

## Publish Files

<Tabs items={["Markdown", "Prototype"]}>
  <Tab>
    ```bash
    artifacts push --owner alice --project-slug default --file ./report.md
    ```
  </Tab>
  <Tab>
    ```bash
    artifacts push --owner alice --project-slug default --file ./prototype.tsx --title "Prototype v1" --private
    ```
  </Tab>
</Tabs>

`push` is the easiest path for agents because it avoids hand-building JSON request bodies.

## Output Contract

- JSON by default when stdout is not a TTY.
- Human-readable text in interactive terminals.
- Successful data on stdout and errors on stderr.
- `--ndjson` for list streams.
- Stable exit codes for invalid requests, not found, forbidden/auth, conflict, and retryable network errors.
